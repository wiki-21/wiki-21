文件分类整理指南

很多人都有随手把临时文件丢在电脑桌面上的“老毛病”，使用没过几天桌面就并各种各样的文件占满了，年复一年，很多文件完成了历史使命，都是“超龄”在桌面上服役。这样既不利于自己查找文件，对桌面的美观也是有很大影响的。散乱的桌面会给人一种缺乏条理的感觉，会让人质疑你的条理性。因此，将桌面或者类似下载等文件夹中的文件进行合理的归类，就显得非常重要。

下文就从个人经验出发，介绍一些文件整理过程中经常使用的原则和方法。

简单来说有两个基本原则来分类管理文件：

第一：临时文件需要单独文件夹；

第二：建立用于长久存储的多层级分类文件夹；

1. 一级目录：磁盘根目录

macOS 等类 Unix 系统的用户一般不会接触到磁盘根目录，常用文件的存储一般都是在 /home/username 文件夹下进行的。因此这部分先来讨论 Windows 系统。

以 Windows 系统为例，虽然现在有许多的磁盘不分区党，但是我们一般都会将硬盘分区，并将 C 盘作为系统盘，那么系统盘就不应作为存储文件的首要磁盘。

建议至少连同 C 盘划分出两个磁盘分区，即 C 盘 + D 盘的方式。其中 C 盘用来作为系统盘，用于常见软件的安装。

关于软件的安装，一般的安装引导程序会将软件默认安装在 Program Files 和 Program Files(x86) 两个文件夹中。但是很多时候我们会对软件中的文件进行一些操作，很容易找不到程序安装文件夹的位置，因此也可以单独划定一个文件夹用来安装第三方软件。关于软件安装的更多内容，可以参看这篇文章。

1.1 系统盘

对于 Windows 系统盘（以 C 盘为例）的根目录，我建议一般不要去新建各种文件夹，以及存放各种文件，保持其安装时的基本结构就可以。

1.2 另一个盘

对于 Windows 的另一个盘（以 D 盘为例）的根目录，我们按照开头提到的基本原则来建立分类文件夹和进行整理。

首先，按照一定的分类原则进行分类。

实际上，很多时候单个文件的分类是相对模糊的，可以属于多个分类。因此，分类之间要有足够的区分度，并且要结合自己的常用文件类型进行分类。

这里提供一个基本的分类思路，也是我正在使用的分类方式。

  1. 为了进行排序，要在文件夹加数字或者字母序号；
  2. 为了避免一些程序的异常，文件夹名以英文为主；

我的次级盘根目录主要有以下分类文件夹：

100 Books （主要存放收集的 PDF、ePub 书籍）

200 Audio （主要存放各种音频文件，包括音乐）

300 Video （主要存放各类视频文件，包括电影）

400 Documents （主要存放除了书籍以外的文档）

500 Pictures （主要存放各种图片，包括照片等）

600 Games （主要存放大型单机游戏）

700 Program （主要存放各类软件安装包、免安装软件）

800 …… （为需要拓展的分类预留序号）

900 Temp （临时文件，需要定期整理至以上各个文件夹中）

可以看到这样的分类模式，是考虑我自身的实际情况的。在 Windows 中如果文件夹中有空格可能会有一些不方便的地方，那么可以用下划线替代。

其次，建立并定期整理临时文件夹。

可以看到，在上述的分类文件夹中，我增加了一个 Temp 临时文件夹，你也可以给它取名 Sundry、Miscellaneous、Drafts 等。对于一些临时需要的文件，比如刚下载的文件、他人传输过来的文件，都可以临时放在这里，避免一股脑丢到桌面。可以在桌面建立这个文件夹的快捷方式。

对临时文件夹中的散乱文件夹要进行定时的整理，不需要的丢进回收站，有明确分类的放到对应的文件夹中。因为在一个盘中操作，传输时间往往可以忽略不计。但手动整理文件可能对很多人来讲也是一大挑战，因此可以借助自动化的工具，Windows 下推荐 DropIt，macOS 下推荐 Hazel 。

1.3 One more thing

Windows 下进行下载时，如果没有经过自定义，那么一般是下载到下载文件夹，位置在 C:\Users\yourname\Downloads 。以 win10 为例，如果在文件管理器中没有隐藏“快速访问”，那么可以将下载文件夹固定到“快速访问”栏。这个下载文件夹，就是绝好的临时文件夹。

2. 次级目录

2.1 二级目录

在根目录下建立一级目录之后，开始考虑各一级目录下的二级目录。

但是不建议设计过多的目录层级，一般主观感受上，以 2~3 级目录最为合适，层级过多会让分类变得混乱，也会让寻找文件花费更多时间，与我们简洁明了的分类原则相背离。

以下以一个名为 Books 的一级目录为例来介绍如何建立二级目录。

之所以以 Books 为例，是因为书籍的分类系统是相对完善的，我们可以借鉴图书馆、档案馆的分类方式来建立二级目录。常见的图书分类法有中国的中图法，这个我们相对熟悉，中国大部分图书馆里 K、H 等开头带一串数字的索书号就是非常好的借鉴分类方式。

为了保证有效性和明了性，我在借鉴图书分类法的基础上，针对自身的情况进行了一些修改，以使得分类适合自己的需求.

2.2 三级目录及多级目录

一般情况下，不建议建立四级及以上的目录。在二级目录中存放的就应该是较为具体的文件、文件夹了。如果确有需要，可以建立更多级的目录。
